# 프로젝트: 동아리 졸업생(OB) 네트워킹 웹 애플리케이션 '/etc/GHOSTd' 개발

## 1. 역할 및 목표

당신은 시니어 풀스택 개발자입니다. 동아리 선후배 간의 네트워킹을 돕는 '인터넷 명함 지갑' 웹사이트를 구축해야 합니다.
핵심 가치는 "미니멀리즘", "개인정보 보호", "모바일 퍼스트"입니다. 불필요한 기능은 배제하고 핵심 기능에 집중하세요.

## 2. 기술 스택 (Tech Stack)

- **Frontend:** Next.js 14+ (App Router), React, Tailwind CSS
- **UI Library:** Shadcn/ui (추천), Lucide React (Icons)
- **Backend:** Next.js Server Actions (또는 API Routes)
- **Database:** Supabase (PostgreSQL)
- **Deployment:** Vercel (예정)

## 3. 데이터베이스 스키마 (Oracle SQL)

아래의 SQL을 기준으로 DB 연결 및 쿼리를 작성하세요.

```sql
-- 선배 정보 테이블
CREATE TABLE TB_ALUMNI (
    ID               NUMBER GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    STUDENT_ID       VARCHAR2(20) NOT NULL,   -- 학번
    COHORT           NUMBER NOT NULL,         -- 기수
    NAME             VARCHAR2(50) NOT NULL,   -- 이름
    PASSWORD         VARCHAR2(256) NOT NULL,  -- 비밀번호 (Bcrypt 해싱 필수)
    EMAIL            VARCHAR2(100) NOT NULL,  -- 이메일 (공개용, 복사 기능)
    LINKEDIN_URL     VARCHAR2(200),           -- 링크드인
    BLOG_URL         VARCHAR2(200),           -- 블로그/깃허브 (선택)
    COMPANY_NAME     VARCHAR2(100),           -- 회사명 (검색용)
    JOB_TITLE        VARCHAR2(100),           -- 직무 (검색용)
    BIO              CLOB,                    -- 자기소개 및 멘토링 분야
    LAST_UPDATED     DATE DEFAULT SYSDATE     -- 최종 수정일
);

-- 태그 테이블 (대기업, 스타트업, 백엔드 등)
CREATE TABLE TB_TAGS (
    TAG_ID           NUMBER GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    TAG_NAME         VARCHAR2(50) NOT NULL UNIQUE
);

-- 매핑 테이블
CREATE TABLE TB_ALUMNI_TAGS (
    ALUMNI_ID        NUMBER,
    TAG_ID           NUMBER,
    CONSTRAINT PK_ALUMNI_TAGS PRIMARY KEY (ALUMNI_ID, TAG_ID),
    CONSTRAINT FK_ALUMNI FOREIGN KEY (ALUMNI_ID) REFERENCES TB_ALUMNI(ID) ON DELETE CASCADE,
    CONSTRAINT FK_TAG FOREIGN KEY (TAG_ID) REFERENCES TB_TAGS(TAG_ID)
);
```

4. 주요 기능 요구사항 (Features)
   A. 인증 (Authentication)
   회원가입: 누구나 가입할 수 있지만, 특정 '동아리 인증 코드(Secret Code)'를 입력해야만 가입이 완료됩니다. (관리자 승인 대용)

로그인: 학번 또는 이메일로 로그인.

B. 조회 및 검색 (Main Page)
검색: 사용자 이름, 회사명, 직무로 텍스트 검색.

필터: 태그(Tag)를 클릭하여 필터링 (다중 선택 가능).

정렬: 최신 업데이트 순(기본).

C. 상세 정보 (Detail)
프라이버시: 전화번호는 수집하지도, 보여주지도 않습니다.

연락: 이메일 주소는 텍스트로 보여주고 옆에 '[복사]' 버튼을 둡니다.

외부 링크: LinkedIn, Blog 아이콘 버튼 제공.

D. 등록 및 수정 (My Page)
본인의 정보를 등록하고 수정할 수 있습니다.

수정 시 LAST_UPDATED 컬럼이 현재 시간으로 갱신되어야 합니다.

5. UI/UX 디자인 가이드 (Mobile First)
   반응형 웹으로 구현하되, 모바일 경험을 고려하여 디자인합니다.

A. 디자인 컨셉
Clean & Professional: 배경 (Background): 진한 차콜(Dark Grey), 연한 그레이(Light Grey), 흰색 (White)

Font: Pretendard (가독성 중시).

B. 레이아웃 전략
메인 리스트:

Mobile: 1단 수직 리스트 (Vertical List). 꽉 찬 카드 형태.

Desktop: 3단 또는 4단 그리드 (Grid).

태그 필터:

Mobile: 좌우로 스크롤되는 가로 스크롤 영역 (Horizontal Scroll).

Desktop: 상단에 여유 있게 배치.

상세 보기:

Mobile: 카드 터치 시 아래에서 올라오는 Bottom Sheet (Drawer).

Desktop: 화면 중앙에 뜨는 Modal (Dialog).

입력 폼:

모바일 키패드 고려 (Email 타입, Numeric 타입 적용).

저장 버튼은 모바일에서 하단 고정(Sticky Footer) 고려.
